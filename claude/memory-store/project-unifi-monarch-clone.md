**Cloning technique that works (faithful, not eyeballed):** drive the user's logged-in Chrome via `browser-harness` and extract Monarch's live `getComputedStyle` numbers, then match them. This caught real gaps (hero 680→800px, `overflow-visible`, headings weight 500 vs Monarch's 350, body leading 1.625 vs 1.4). `monolith` (brew) is installed as an offline page-archive fallback. Monarch's measured type: h1 48px/350/-0.067em, h2 40px/350/-0.06em, eyebrow 14px/500/0.05em, card shadow `0 4px 6px -1px / 0 2px 4px -2px rgba(34,32,29,0.075)`, nav 108px.

**Hero is a Veo 3.1 video, NOT a static image** (don't swap to static). Gen pipeline in `docs/design/monarch-teardown/_kit/`: `gen-ref.py` drives ChatGPT Image 2.0 for the anchor still (run via `browser-harness`), then `veo-gen.py` does image→video. Veo billing is now ACTIVE. Run veo-gen via `uv run --with google-genai python3 veo-gen.py` (the SDK isn't in the harness env, and veo-gen needs plain python not browser-harness).

**Veo pillarbox gotcha (cost time twice):** Veo nests a 3:2 desktop anchor / 2:3 mobile anchor inside a 16:9 / 9:16 frame with ~100px BLACK BARS each side. Always crop after generating: desktop `crop=1080:720:100:0`, mobile `crop=720:1080:0:100`. Verify edges aren't black before encoding.

**Anti-AI-look:** gpt-image-1's plastic sheen is beaten by documentary-grade prompts (named camera + Kodak Portra 400 + imperfection cues: visible pores/flyaway hair/film grain + hard anti-CGI negatives). Real product UI (live `AppMockup` component) as section visuals beats compositing UI into generated lifestyle scenes — that was the worst AI tell.
