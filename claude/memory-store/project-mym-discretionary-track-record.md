The forward-testing journal is split by ORIGIN so the founder's discretionary edge can be studied apart from the full-auto "box." Built 2026-07-20.

**Separation rule (CORRECTED 2026-07-20, coded — do not reclassify without re-verifying against mym-autotrader's own lane_registry.py):** a trade is MANUAL (founder hand-called) iff `instrumentRoot == "YM"` AND (blank `entryName` OR `entryName`/`strategyTag` matches `(us30|relay|manual)` case-insensitive). This matches mym-autotrader's own production classifier (`service/lane_registry.py` `_MANUAL_RX`, asserted by `tests/test_lane_registry.py::test_relay_and_short2_entry_names_go_to_relay_bucket`) and `forensics/forward-vs-backtest-scorecard.md`, which bucket `us30-relay-*` AND `us30-short2-*` as founder relay tickets — NOT bot strategies, even though `us30-short2` looks bot-tagged. An earlier version of this rule (blank-entryName only) undercounted the founder's book by $4,700 and wrongly counted it as box auto — corrected same-session. Everything else (all non-YM instruments) is THE BOX.

**Capture the read at call-time:** `brain-call <market> <side> <entry> "<your read>" [tags...]` (→ `~/local/bin/brain-call`, wraps `~/brain-os/study/discretionary.py log`). Logs the WHY so the brain can learn which setups pay. Reconciled to outcome daily. For YM it ALSO prints the level's historical reaction strength instantly (see below) before the log saves.

**Level Radar (live, built 2026-07-20):** `~/brain-os/brains/ym/engines/level_snapshot.py [--price P | live]` scores how strong a YM price level is — weight = distinct days (of the last 25) it was a pivot high/low, tier WEAK(<5)/DECENT(5-7)/STRONGEST(≥8) — using the leak-free 25-day reactive-zone map ([[claim-winning-fade-was-strongest-reactive-zone]]). Also computes a live TOUCH ALERT: CLEAR (>20pt from any zone) / NEAR (≤20pt, heads-up) / TOUCHING (≤5pt, at it now) — a STRONGEST+TOUCHING fires a pulsing red alarm on the dashboard. App: `scripts/export_levels.py` → `data/levels.json`, rendered as the "Level Radar" panel atop the Forward tab, refreshed every 15min by the existing `com.area61.nexus-refresh` launchd job (now also refreshes `forward` + `levels`). Fail-soft: a failed live fetch keeps the last-good snapshot (marked stale) rather than blanking the gauge.

**Files:**
- `~/brain-os/study/discretionary.py` — `log` / `reconcile <journal.json>` / `study [--emit]`. Ledger `discretionary_calls.jsonl` is PERMANENT + append-only (grows call-by-call; unlike the 60-day journal window). Maturity gate: `forming (<10 calls)` → `evaluable (>=10)`, then compare discretionary vs box per-market.
- `~/brain-os/study/forward_scoreboard.py <journal.json>` — box-vs-manual scoreboard.
- `~/brain-os/study/box_execution.py <journal.json> [--emit]` — loss attribution: classifies each box loss as ENTRY (mfe<|loss|, never went favorable → wrong entry; a wider stop only loses more) vs MANAGEMENT (mfe>=|loss|, gave back a real gain). Verdict (corrected, YM excluded): 90% of box losses are ENTRY-driven ($6.99k vs $0.73k give-back), mean entry-eff 0.39 — the box's lever is entry filtering, NOT exit tuning. HE (eff 0.00), MGC (0.02) = zero entry edge, disable; MES (0.31, -$788) = re-signal. Claim `claim-box-losses-are-entry-driven` (0.8). Uses journal MFE/MAE/efficiency (fully populated; `isManual` is USELESS — 0 flagged).
- `~/brain-os/study/dead_zone.py <journal.json> [--emit]` — per-sleeve ENTRY dead-zone finder: zero-winner time-of-day / day-of-week windows, gated by binomial significance `(1-base_win)^n < 0.05` (kills low-base-rate noise — MES/MYM correctly rejected, only MNQ survives). Surviving prescription (FORWARD-TEST, not law): **skip MNQ entries at 08:00 ET + Friday → +$914 (26%), win 38%→53%, PF 1.84→2.80, 0 winners lost.** Claim `claim-mnq-entry-dead-zones` (0.65) with falsifier "a winner appears in a dead-zone." The box's #1 lever is entry filtering (see [[claim-box-losses-are-entry-driven]]).
- App: Area61 NEXUS **Forward** tab (`src/renderer/surfaces/forward.js`); `scripts/export_forward.py` prefers the permanent `discretionary.json` track record over the window. Panels: Level Radar (live), box keep/cut chart, "Why the box loses" (loss attribution + disable candidates + dead-zone prescriptions), **Backtest Parity** (real, wired 2026-07-20), "Your calls" track record.
- `~/brain-os/study/entry_context.py <journal.json> <ROOT> [--emit]` — leak-free market-state (trend/VWAP-dist/vol-regime/rangepos) reconstructed at each entry from yfinance 1m bars, up to entry time only (no look-ahead). Found MNQ's box wins are never counter-trend (0.00) vs 45% of its entry-fails (skip-CT lift +$1,466, 0 winners lost) — MNQ-specific, does NOT generalize (MES shows the opposite lean on a thin sample, see [[claim-mnq-trends-founder-fades-scoped]]). For YM it characterizes the founder's calls' market context (trend/rangepos at each fade).
- **Backtest Parity (real, wired 2026-07-20):** mym-autotrader's OWN `service/fidelity_scorecard.py` (self-contained, pulls its own CrossTrade bearer token from `~/.claude.json`, live HTTP fetch — no MCP session needed) matches real historical backtest OOS PF (`vault/STRATEGY-REGISTRY.md`) against actual live/demo forward fills, per (strategy, market) — 16 tracked cells. Regenerates `forensics/forward-vs-backtest-scorecard.md`; `scripts/export_parity.py` (area61 app) parses that markdown table (NEVER imports mym-autotrader service code directly — same read-only-on-plaintext rule as every other exporter here) into `data/parity.json`. Gotcha: Python's `json.dump` writes bare `Infinity` for `float('inf')` PF values, which is NOT valid JSON and breaks `JSON.parse` in Electron — encode as the string `"inf"` instead. Wired into the 15-min refresh loop (`refresh.sh` now regenerates the mym-autotrader report first, fail-soft — a failed live fetch just leaves the last-good report in place, per the source script's own H105 discipline). This is DIFFERENT from the Sim101 tab (that one is forward data mislabeled as backtest, see [[backtest-tab-is-not-a-backtest]]) — this one is genuinely a historical backtest baseline. Every cell is N<20 (mym-autotrader's own read) so PF is noise at this sample size; only 1 cell is currently `DIVERGING` (Area61Precision_v2 on YM).
- Autopilot: `daily_study.sh` deterministic tail reconciles + studies + re-exports every run (weekdays 16:35 ET, `com.jonathan.brain-trade-study`).
