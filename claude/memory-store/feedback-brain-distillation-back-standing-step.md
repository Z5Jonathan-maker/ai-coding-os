**Founder directive (2026-07-17):** "Our master brain needs to keep growing and it needs to keep actually making you smarter. This should've been part of all the loop reasoning work you did today."

**The gap it names (own it):** committing vault DOCS persists facts *in-repo* but is an ARCHIVE, not a brain that changes future reasoning. "Grew a file" != "got smarter." A whole day of loop work (cracks, gauntlet audits, vendor forward-tests, Providence mine) produced major durable knowledge that mostly lived in one-off commits + chat — the session-loaded layers (auto-memory + MEMORY.md, repo decisions.md/SESSION-HANDOFF/agent-ledger, wiki decision-rules) and the encode-as-RULE step were skipped.

**HARD BEHAVIORAL RULE — every non-trivial cycle ENDS with distillation-back:**
1. Extract the durable learnings from the cycle.
2. Encode each as an actionable RULE in a session-loaded layer — phrased "when X, do Y" NOT "we did Z". Archive-phrasing = failure.
3. Verify it's behavior-changing (would it alter a future decision?), not archival.
4. Route by layer: cross-session facts/rules -> auto-memory file + MEMORY.md pointer; methodology corpus -> `mega-brain-ingest` / `cc-reflect optimization`; repo continuity -> the AI-SESSION-START handoff (decisions.md + SESSION-HANDOFF + agent-ledger) — in a repo, THAT handoff IS the repo-brain-growth step, do it, don't just commit vault docs.
5. Wire this phase into the capstone/autonomous-loop reasoning so it runs automatically, not reminder-driven.

**This is the COMPLEMENT to [[feedback-corpus-as-brain-standing-directive]]:** that one = intake (pull corpus-first for hypotheses); this one = distillation-back (write learnings home as rules). Together they close the loop: corpus grows -> better hypotheses -> tested -> results distilled back as rules -> brain measurably smarter. Honest mechanism: the model doesn't auto-learn between sessions; "smarter" is real at the SYSTEM level, carried by memory — so skipping the write-back literally forfeits the smarter.
