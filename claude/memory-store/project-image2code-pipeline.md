Built 2026-06-07: image→code as a first-class capability in the AI-SYSTEM-V2 router, productizing the field's #1 gap (nobody ships a reusable closed pixel-diff refine loop). Pattern researched via the `image2code-playbook-hunt` workflow (10 adversarially-verified playbooks); canonical = Saqoosha's Pencil→Claude Code→PIL-diff loop (94.8% pixel match, the only metrics-bearing one).

**Artifacts (all additive to the locked 4-tier router; no new platform):**
- `~/.claude/scripts/i2c-diff.py` — pixel-diff + SSIM + per-region scoring + worst-region heatmap. PIL+numpy only (SSIM hand-rolled, no skimage).
- `~/code/projects/cc-router/lib/image2code-pipeline.cjs` — orchestrator: generate(S0)→transcribe(S1)→render(S2)→diff(S3)→fix-loop(S4). Resolution-pinned (default 1440×900). Stop reasons: target_reached / plateau / plateau_at_ceiling / max_passes. Dry-run by default; `--live` fires real bridge+agent+screenshots. CLI: `node image2code-pipeline.cjs "<prompt>" <projectDir> --live --aspect web --target 96`.
- `~/AI-SYSTEM-V2/router/capability-registry.json` — `image_to_code` capability (claude primary, kimi fallback) + `mockup_to_app` 5-stage workflow.
- routing: `purpose:image_to_code` → precision/Claude (deterministic). Added to `tiers/precision.cjs` + registered in `validate.cjs` VALID_PURPOSES and VALID_FLAGS (the startup integrity check fails loud if missing — required for any new purpose/flag).
- dashboard: "Image 2.0 → Code" card + `POST /api/image2code` (dry-run preview) in `~/AI-SYSTEM-V2/dashboard/index.html` + `scripts/dashboard-server.cjs` (served on :8787; NOTE the served index is `AI-SYSTEM-V2/dashboard/index.html`, NOT the AI-HQ one).

**Key facts:** both Claude+Kimi router lanes are CLI-agentic (`spawn`), so image grounding = pass the frame's FILE PATH in the prompt and let the CLI `Read` it — no base64 vision-API plumbing. Resolution pinning (export res == screenshot viewport) is the load-bearing invariant. ~95% is the irreducible text/font-render ceiling vs Chrome — don't chase below it.

**Known limitation:** NL phrasings ("image to code") route non-deterministically (embedding/codex classifiers intercept before precision's last-in-REGEX_ORDER regex). Doesn't matter — pipeline + dashboard always pass the explicit purpose. Future fix: own early-ordered tier.

HARD RULES learned (each from a live regression): OVERLAY never REPLACE (removing an SVG broke the JS that populated 5 panels); element-precise edits only (assert div-count unchanged — "replace inner content" over-deletes siblings); monotonic gate catches all regressions; crop extract-assets BELOW the panel header (avoid duplicated titles). Vision-judge plateaus ~88 cosmetic-only on diffusion targets and is noisy ±few pts — don't chase last 2 points. Fidelity ceiling: conventional UI ~88 one-shot; dense data-viz/3D ~88 ONLY via decompose-route+extract.
