THE VAULT = the mym-autotrader dashboard elevated into a trading-business OS (founder charter 2026-07-05, filed verbatim in `mym-autotrader/docs/DESIGN-VAULT.md`). Four products under one roof: **Command** (live ops/risk/health) · **Fleet** (accounts/payout pipeline) · **Vault Academy** (interactive strategy library) · **Vault Intelligence** (AI analytics). Design law: ELEVATE never redesign; understated luxury + institutional finance (Apple/Linear/Bloomberg/F1); command decisions, not data ("I'm safe" > raw numbers).

**Why:** the founder's end goal is a platform someone with 50-100 funded accounts couldn't trade without — not a personal dashboard.

**How to apply:**
- HONESTY RAIL (binding): charter's illustrative numbers (18 accounts, $2.45M, "Siege"/"Liquidity Trap", Apex/Topstep) are vision-scale examples — NEVER render them. Build structures, populate only from real feeds (config/accounts.json, floor_state, governor, dashboard-state). Fields with no feed (health-score model, recommended contracts, regime, confidence) → omit, never guess.
- TRAP: `docs/dashboard.html` is GENERATED by `service/dashboard.py` AND git-flagged **skip-worktree** (`git ls-files -v` → `S`) — hand-edits are invisible to git and wiped on regen. Always edit the generator (plain-string constants injected into the f-string template, the QA_CSS/QA_JS pattern). Same discipline: `dashboard-public/` is a deploy artifact (`service/deploy_dashboard.sh` wipes + regenerates + AES-encrypts it).
- Waves 2-3 shipped in-generator 2026-07-05 (status rail, intelligence panel, motion/count-up, account-health block, War Room, window countdown). Wave 4 backlog in [[project-mym-autotrader-assistant]] repo's SESSION-HANDOFF.md.
