NOTE (verified 2026-07-20): the earlier audit claim that `secret-paste-guard`+`loop-guard` are UNWIRED is FALSE — both are wired + live (secret-paste-guard→UserPromptSubmit, loop-guard→PreToolUse; no override in settings.local.json). Real security gap = 4 plaintext `Bearer` tokens in `~/.claude.json` (crosstrade order-capable ×2, crosstrade-oauth, obsidian), likely single-copy (not found in Keychain under obvious names). Control plane: cc-router mature (tiers/circuit-breakers/budget/receipts) but multi-agent layer has NO shared queue/lease/lock.

**80/20 roadmap:** (1) move 4 plaintext Bearers→Keychain via `${VAR}` header expansion (CONFIRMED supported CC 2.1.x, code.claude.com/docs/en/mcp.md; env must be exported in shell that launches `claude`; unset→literal `${VAR}` sent) — OPERATOR-RUN (order-capable crosstrade auth; needs CC restart to verify) + CI to keep the already-wired guards wired; (2) lift CANON guard OS-wide [days]; (3) consolidate memory→one event-log substrate + backfill, dual-read [~1wk]; (4) hybrid+temporal retrieval [~1wk]. Steps 1-4 = fragmented+grep → one governed substrate w/ temporal recall.
